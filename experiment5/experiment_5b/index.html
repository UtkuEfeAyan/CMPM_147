Description

For this experiment, I wanted to make a generator that feels like an image is slowly forming over time—not instantly shown, but built through layers of mutated shapes. My idea was to start with large, soft circles and then keep adding smaller, more specific ones as time passed. Basically, create something that slowly becomes more recognizable and detailed. But then I realized the assignment wanted more user control and mutation-based interaction, so I made two versions: one that's more straightforward and fits the assignment exactly (experiment_5a), and another (experiment_5b) which this one is experiment_5b. which mushc more of waht i wanted but might violate the fps limit rule. The final version is super customizable and lets the user control how quickly the image evolves, how soft or sharp it looks, and how chaotic the generation feels.

Technical

I built a visual mutation system that draws transparent circles onto a canvas to try to match a target image. You can control mutation cycle speed, number of new circles per cycle, min and max radius of the circles, and their alpha (transparency). All of this is controlled through sliders and number inputs, so the viewer shapes the process directly. The circle data is stored and mutated across time, and every few frames (depending on the threshold you set), new circles are added based on your parameters. There's also a live circle counter and score tracking (based on color matching error to the target image). I implemented safety limits to prevent the FPS from dropping too hard, and added a bunch of toggles to make the generator more modular and tweakable. Honestly it’s kind of like a pointillist version of an evolutionary algorithm but with visual feedback and manual control.

Reflection

This was the experiment I spent the most time on by far. It started simple, but I kept wanting to add more to it—more control, more feedback, more personality. I liked the idea of the user being able to tune how the image “emerges,” and I ended up with something that felt really satisfying to watch. It wasn't easy though—there were a bunch of issues with performance and debugging visual glitches when too many circles were added, and some math stuff with how alpha blended. But in the end, it was worth it. I had fun trying to make something that feels alive and shaped by the viewer. It’s also one of those pieces where the randomness feels meaningful because you're giving it a shape to follow, not just letting it go completely wild. And yeah, I might have gone overboard with how many tweakable variables there are—but that’s the fun part.
